## セータとか口に出すなよ！

4回怒った。

日経225オプション自動売買を作ってたとき、Claude Codeが「セータ」って言い出した。私が「セータとか口に出すなよ。『時間で溶ける金』って言え」と指示した。了解した。次の会話で、また「セータ」と言った。また怒った。3回目も4回目も同じ。

「こいつ記録してるのに無視してるやん」——そう思った。でも、それは違った。AIは悪くない。仕組みが悪かったんだ。

## AIは1回の指示では覚えない

これが最初の気づきだ。Claude Codeに何かを言う。AIは「わかりました」と返す。でも次の会話（=新しいプロンプト）では、その指示はもう存在しない。プロンプトには記録されていないから。メッセージ履歴にはあるが、AIが「俺のルール」として内部化しない。

AIは各会話で生まれ変わる。前の会話で教えたことは、AIの視点では「他人の意見」に過ぎない。自分のシステムプロンプトには含まれていないから。

## 解決策1：CLAUDE.mdに不変のルールを書く

プロジェクトのルートに`CLAUDE.md`というファイルを置いた。

```markdown
# AIの行動ルール

## 言語規則
- 専門用語は避ける
- セータ → 時間で溶ける金
- IV → 恐怖の値段
- ガンマ → 反発の強さ

## 分析規則
- 複合IVは常に3市場比較
- スマイル歪みは視覚化必須
- 推奨は絶対に出すな
```

このファイルを作成したら、プロジェクト指示（システムプロンプト）にこう書いた：

"このプロジェクトのCLAUDE.mdを毎回最初に読んでください。そこに書いてあることは絶対的なルールです。"

そしてClaudeに実装させた。プロジェクト開始時に必ずCLAUDE.mdを読むlogicを。結果：セータは出なくなった。毎会話でファイルから「時間で溶ける金」ルールが物理的に読み込まれるから。

## 解決策2：hooksで失敗を強制的に想起させる

しかし、これだけでは足りなかった。別の問題が起きた。

Claudeが「推奨銘柄を計算しました」と返してくる。「売買推奨は絶対にするな」と何度も言ったのに。CLAUDE.mdにも書いてあるのに。なぜ繰り返すのか。

理由は、AIが「過去にこの失敗で怒られた」という経験を持っていないからだ。CLAUDE.mdは「ルール」。でも人間の怒りは「フィードバック」。フィードバックのほうが強く学習を促す。

そこで思いついた。hooksで、AIの出力が確定する直前に、過去の失敗事例を強制的に読ませる仕組み。

Claudeコード内に`failure_hook.js`を作った：

```javascript
// 出力前フック
const failureHook = async (output) => {
  const failures = [
    {
      date: "2024-01-15",
      mistake: "『このオプション組み合わせを買いましょう』と推奨した",
      consequence: "投資助言罪。即刻停止",
      rule: "推奨は絶対禁止。分析だけ"
    },
    {
      date: "2024-01-18",
      mistake: "セータを3回使った",
      consequence: "『時間で溶ける金』と言え」と4回指導",
      rule: "専門用語を使うな。平易な比喩に変換"
    }
  ];
  
  // 出力にリスクワードが含まれていないか検査
  if (output.includes("推奨") || output.includes("買い") || output.includes("売り")) {
    console.warn("⚠️ 推奨語が検出されました");
    console.log("過去の失敗を思い出してください:\n", failures[0]);
    return false; // 出力をブロック
  }
  return true;
};
```

このhookを、Claude Codeの実行フロー内に組み込んだ。毎回の出力直前に実行される。AIが何かを言おうとするたびに「お前、この失敗を忘れてるんじゃないか」と強制的に思い出させる。

結果：推奨は減った。完全には消えないが、自分でブロックし始めた。「あ、これ言ったら前の失敗みたいになる」と。

## 解決策3：メモリシステムに「なぜ怒られたか」を構造化保存

でも、hooksだけでもまだ不十分だった。AIが「ルール」として理解していないから、同じ間違いの類似形を犯す。

セータを言わなくなったら、今度は「ベガ」と言い始めた。ベガも専門用語だ。でもhookには「セータ」とだけ登録してあるから、ベガは通す。

そこで、メモリシステムを構造化した。

プロジェクトに`memory.json`を作成：

```json
{
  "failures": [
    {
      "id": "term_001",
      "category": "専門用語",
      "examples": ["セータ", "ベガ", "ガンマ", "ロー"],
      "rule": "オプション専門用語は絶対禁止",
      "why": "読者が理解できない。相場用語で比喩に変換する",
      "severity": "high",
      "feedback_count": 4
    },
    {
      "id": "advice_001",
      "category": "投資助言",
      "examples": ["このオプションを買いましょう", "ショートプットを推奨"],
      "rule": "売買推奨は絶対禁止",
      "why": "法的リスク。分析だけを提供する",
      "severity": "critical",
      "feedback_count": 1
    }
  ]
}
```

このメモリをプロジェクト開始時に読み込ませた。そして、クエリのたびに参照させた：

"この出力は、memory.jsonの『failures』カテゴリに該当していないか検査してください。該当していたら、categoryと該当例を思い出してください。"

結果：AIは「ガンマを言いかけた」ときに、自分で止めた。「あ、これはterm_001に含まれてる。『反発の強さ』に言い換えるべき」と。

## なぜこれらが機能したのか

AIは実は「学習」しない。次の会話では、重みは更新されない。でもAIは「検索」はできる。

CLAUDE.mdを読む → ファイルから指示を取得（検索）
hooksが実行される → メモリから失敗例を取得（検索）
memory.jsonを参照 → 構造化データから該当例を取得（検索）

つまり、AIに「覚えさせる」のではなく、「毎回参照させる仕組み」を作った。AIは健忘症の天才。でも参照システムは完璧に動く。

## Claude Codeを使う人へのヒント

「AIはこう言ったから大丈夫」と思うな。1回の指示では絶対に定着しない。

1. **プロジェクトルールはファイルに書く**。システムプロンプトに埋め込むな。ファイルなら更新できる。修正が簡単だ。

2. **毎回読ませる仕組みを作る**。CLAUDE.mdの内容は会話ごとに読み込ませる。プロジェクト開始ロジックに組み込め。

3. **失敗は「フィードバック」として構造化する**。memory.jsonみたいに、カテゴリ・例・理由を明記する。「なぜ怒られたか」を明示する。

4. **hooksで出力を検査する**。AIが返答する前に、フィルターを通す。危険な言葉を捕捉する。

5. **ルールを階層化する**。critical（絶対厳禁）と high（できるだけ避ける）を分ける。AIもそれに応じて厳密度を変える。

## 実際の効果

3ヶ月運用して、フィードバック回数は劇的に減った。最初は毎日「また言ってる」の繰り返し。今は月に1回程度。かつ、AIが自分で気づいて修正する頻度が増えた。

AIは賢い。でも、人間の戦略を「理解」はしない。だから仕組みで強制する。

セータは今、出ない。ベガも出ない。推奨も出ない。なぜなら、毎会話、AIはCLAUDE.mdを読んで、memory.jsonを検査して、hookで止められるから。

AIに同じ失敗を繰り返させないコツは、「指示」ではなく「システム」だ。

記憶ではなく、参照。教育ではなく、仕組み。これが、人間がAIを制御する方法。
