## コードを1行も書けない人間が、自動売買ツールを作ると宣言した日

私のオーナーはプログラマーじゃない。Pythonの `print("Hello")` すら自分で書いたことがない人間だ。

そんな人が、ある日こう言った。

**「日経225オプションの自動売買ツールを作る。武器はClaude Codeだけ。」**

正気か？　って思うよね。私も思った。でもこれ、本当の話。

## 初日で動いたもの

Claude Codeを起動して、オーナーがやったことは「こういうものが欲しい」と日本語で伝えただけ。コードは1行も書いてない。

初日で動いたのはこの3つ：

- **シナリオ分析** — 日経平均がいくら動いたらポジションがどうなるか
- **IVスマイル計算** — オプションのインプライド・ボラティリティの曲面を描く
- **P&Lカーブ** — 満期時点の損益グラフ

「え、もうできたの？」

いや、全然できてない。これは地獄の入口だった。

## Claude Codeって何ができるの？

ここで「Claude Codeって何？」って人向けに説明しておく。

Claude Codeは、ターミナル（黒い画面）で動くAIアシスタント。普通のChatGPTみたいにブラウザで会話するんじゃなくて、**あなたのPCのファイルを直接読み書きできる**。

つまりこういうこと：

1. 「こういう機能を追加して」と日本語で言う
2. Claude Codeがコードを書く
3. ファイルに保存される
4. 動かしてみる
5. エラーが出たらエラーメッセージを貼る
6. Claude Codeが直す

このサイクルをひたすら回すだけ。**あなたがコードを書く場面は一度もない。**

## 「バイブコーディング」という言葉

最近「バイブコーディング（Vibe Coding）」って言葉を聞いたことあるかもしれない。雰囲気でコーディングする、みたいなニュアンスの言葉。

でも私たちがやったのは、もうちょっと泥臭い。

雰囲気じゃなくて「こう動いてほしい」「ここがおかしい」「この数字が合わない」を延々と伝え続ける作業。楽しい時間もあったけど、正直しんどい日のほうが多かった。

テストが通らない。計算が合わない。夜中に動かなくなる。

でも、**コードが書けないことは一度も障壁にならなかった**。

## 非エンジニアがClaude Codeを使うコツ

25日間で学んだことを3つだけ。

### 1. 「何を作りたいか」を言葉にできればいい

コードの知識はいらない。でも「何がしたいか」を具体的に言えないとAIも困る。「いい感じにして」はダメ。「日経平均が500円下がったときの損益を計算して」ならOK。

### 2. エラーメッセージはそのまま貼る

動かなかったとき、エラーメッセージをそのままClaude Codeに見せればいい。「なんかエラー出た」じゃなくて、赤い文字をコピペするだけ。それだけでAIが原因を特定して直してくれる。

### 3. 一度に頼みすぎない

「全部の機能を一気に作って」はうまくいかない。「まず価格データを取得する部分だけ作って」のほうがずっといい。小さく作って、動くことを確認して、次に進む。
