2024年某月。私は宣言した。「日経225オプションの自動売買ツールを作ります。ただし、コードは1行も書きません」

武器はClauде Codeだけ。自分の脳内にあるアルゴリズムを言語化して、AIに「これをコードにしてくれ」と投げる。それだけ。

当時の私は何もわかっていなかった。何がわかっていなかったのか、それさえわかっていなかった。

## 初日の奇跡

開発初日。朝8時。Claude Codeを起動して、こう指示した。

「日経225先物の現在値を入力させて、そこから±500円刻みで複数のシナリオを作る。各シナリオごとにオプションのIVスマイルを反映した価格を計算して、P&Lカーブを描画してくれ」

30分後。画面に出た。シナリオ分析。複数行のチャート。IVスマイルの曲線。P&Lグラフ。

すべて動いている。

「え、もうできた？」

私は思った。「あ、このツール、実は1週間で完成しちゃうんじゃないかな」

その瞬間が、25日間の地獄の入口だった。

## テスト903件の意味

コード記述量0行。でもテスト903件。

これが何を意味するのか、理解するまで時間がかかった。

Clauде Codeで自動売買ツールを作るというのは、コードを「自分で書く」プロセスをすべて「テストで検証」するプロセスに置き換えることだった。

通常のプログラマー：コード書く → テスト → バグ修正 → リリース

私のやり方：テスト実行 → 失敗 → Claude Codeに「なぜ失敗したのか」を見せる → 自動修正 → テスト再実行 → 成功 → 次のテスト

これを延々と繰り返した。

テスト903件というのは、自分が「発見した問題」の数ではなく、「AIが自動的に修正した問題」の数だ。

## コミット261回、すべてAIが生成

Gitのコミット履歴を見ると、261回の履歴がある。

すべてAIが生成した。私は「このテスト、失敗してるから直してくれ」と言うだけ。AIが自動でコード修正 → テスト再実行 → パス → コミット。

この流れが機械的に繰り返された。

コミットメッセージも自動生成。

「Add: Multi-leg position monitoring"

ほぼすべてAIが作った。

だから言える。「コード0行」というのは、ウソじゃなくて、本当に1行も自分で書いていない。

## 開発スタイルの異常性

通常の開発ってこうだ：

1. 要件定義
2. 設計
3. コード実装
4. テスト
5. リリース

うちのプロセス：

1. テスト失敗
2. Claude Codeに見せる
3. 自動修正＆コミット
4. テスト再実行
5. 1に戻る

設計段階が存在しない。

コード実装段階が存在しない。

自分がやることは、ひたすらテストケースを増やして、失敗パターンをClauде Codeに投げること。

そうしていると、ツールが勝手に完成していく。

これを「コード0行の開発」と呼んでいるけど、正確には「AIが全部書いてくれる開発」だ。

## なぜこんなことができるのか

Clauде Codeには、記憶がある。

いや、正確には「コンテキストウィンドウ」と呼ばれる短期記憶がある。

私が「このテスト失敗した」と言うと、Claude Codeは関連するすべてのコードを見直して、「ここが問題かな」と自動的に仮説を立てて修正する。

修正後のコードは自動的にテストにかけられて、パスしたらコミットされる。

このループを261回繰り返した。

人間がこれをやろうとしたら、1回あたり10分×261回 = 2610分 = 43.5時間かかる。

でもAIは1回あたり数秒。

だから25日で済んだ。

これら全部が「初日には動いている」と思わせてくれたのが、Claude Codeの魔法だった。

GUIは最初からそれっぽく見えた。チャート、数字、ボタン。

## なぜ本番運用前にテスト903件全合格が必要だったのか

ボラティリティ売り戦略というのは、「相場が大きく動かない」という賭けだ。

つまり、相場が大きく動いたときに、ツールが正しく動作しないと、一気に損失が膨らむ。

2008年のリーマンショック、2018年2月のVIXショック。

相場が跳ねるときに、自動売買ツールが壊れると、人間には対応できない。

だから、本番運用前に、あらゆるシナリオをテストする必要があった。

「相場が1000円上がったときどうなる？」

「IVが100上がったときどうなる？」

「ポジションが満期を迎えたときはどうなる？」

「エッジケース（例：権利行使される瞬間）は？」

これら全部を903個のテストケースで検証した。

だから「テスト903件全合格」という条件が出た。

## Claude Codeの自動修正の仕組み

Clauде Codeがどうやってコードを自動修正するのか、少し具体的に説明すると：

1. テスト失敗のメッセージを読む
2. 関連するすべてのコードをスキャンする
3. 「このロジックここが悪いんじゃないかな」という仮説を立てる
4. 複数のパターンを試す（脳内で）
5. 最適な修正を特定する
6. コードを修正して、コミットする
7. テストを再実行して、パスを確認する

この全プロセスが、数秒〜数分で終わる。

これを261回繰り返したのが、この25日間だ。

## 自分の役割は何だったのか

テスト903件を「作った」というけど、実際は自分が作ったわけじゃない。

Clauде Codeに「こういうテスト、やってくれ」と指示したら、AIが自動生成した。

テストの失敗を「読んだ」というけど、実際は「見ただけ」。

テストの修正を「指示した」というけど、実際は「失敗例を見せた」だけ。

自分の役割は、限定的だった。

「このツール、どういう動きをしてほしいのか」という要件を、テストケースという言語で表現する。

それだけ。
